# Функционални изисквания
- Регистрация с Име, Емейл, ЕГН и Парола.
- Автоматично разпознаване на роля по домейн: `@ucb.bg` → `specialist`; `@supervisor.ucb.bg` → `supervisor`; всичко останало → `customer`.
- Клиент: подава жалба (категория, описание, срок за отговор), получава референтен номер и SLA, следи статуса в „Мои жалби".
- Клиент: приема или оспорва отговора на специалиста.
- Специалист: вижда опашка с маршрутизирани жалби по категория, отваря случай, получава AI Copilot чернова.
- Специалист: приема, коригира и изпраща отговора; задейства монетарно действие с флаг `monetary: true` при нужда.
- Ръководител: одобрява или отказва монетарни действия; преглежда оспорени жалби с пълен audit log.
- Референтен номер във формат `CMPL-YYYYMMDD-XXXX` + обявен SLA (3 работни дни).

# Нефункционални изисквания
- Реален мониторинг на статуса (Firestore onSnapshot).
- Адаптивен дизайн (Tailwind CSS).
- Сигурен достъп през Firebase Rules.
- Gemini API за AI Copilot draft (model: `gemini-2.0-flash` или еквивалент).
